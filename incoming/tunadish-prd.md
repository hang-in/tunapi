## 1. 개요

### 배경
tunapi는 Mattermost/Slack/Telegram ↔ AI CLI 브릿지 프로젝트다. 운영하면서 AI 에이전트와만 채팅하는데 범용 채팅앱 백엔드가 과하다는 것을 확인, AI 채팅 전용 경량 클라이언트 프로젝트로 tunadish를 시작한다.

### 목표
- AI CLI 에이전트(claude, gemini, codex)와의 채팅에 최적화된 네이티브 앱
- 인간 간 채팅 불필요, 채팅앱 프로토콜에 맞출 필요 없음
- 어디서든(PC, 모바일) 로컬 서버의 AI CLI를 편하게 제어

### 비목표
- 범용 메신저 기능
- 인간 간 채팅
- 외부 클라우드 의존

---

## 2. 아키텍처

### 전체 구조
```
tunapi (CLI 브릿지, 기존 유지)
  └─ tunadish transport 플러그인 (새 트랜스포트로 추가)
       ↕ WebSocket (JSON-RPC 2.0)
tunadish 클라이언트 (Svelte + Tauri)
```

### 관계
- tunadish는 tunapi의 transport 플러그인으로 동작
- tunapi 코드 수정 없이 entry_point 등록만으로 연결
- 별도 레포로 관리

### 미래 방향
- tunapi Slack 트랜스포트 완료 후 대규모 리팩토링 예정
- 리팩토링 시 공통 코어는 `tunapi-core`로 분리 가능
- tunadish는 이후 `tunapi-core` 의존성으로 교체

| 영역 | 기술 |
|---|---|
| 클라이언트 UI | React + TypeScript |

## 5. 핵심 기능

### 5.1 프로젝트
- 프로젝트 생성 / 관리
- 프로젝트별 에이전트 + 페르소나 + 스킬 바인딩
- 프로젝트 = 채팅 공간의 단위 (Slack 채널 개념)

### 5.2 채팅
- AI 전용 채팅 (claude / gemini / codex)
- 프로젝트별 독립 채팅 공간
- 마크다운 렌더링, 코드블록 처리

### 5.3 브랜치 (서브 대화)
- main 대화에서 브랜치 생성
- 브랜치 실행 방식: 단일 에이전트 / 토론 모드 선택
- 종료 방식:
  - **merge**: AI 정리 텍스트 + 사용자 코멘트 → main 컨텍스트에 append
  - **abandon**: 결론 없이 닫기
- Git 개념 적용: branch / merge / abandon / checkout (과거 분기점 재탐색)

### 5.4 토론 모드
- 브랜치의 실행 방식 옵션 중 하나
- 설정: 주제 / 참여 에이전트 / 역할 및 페르소나 / 발언 순서 / 라운드 수
- 진행: 턴 오더 기반 순차 실행
- 사용자 개입 포인트:
  - 특정 에이전트에게 질문
  - 요약/정리 지시
  - 턴 오더 변경
  - 새 에이전트 투입
- 종료: 결론 정리 + 코멘트 → main에 merge 가능

### 5.5 페르소나
- 프리셋 CRUD
- 에이전트와 독립적으로 존재 (1:1 고정 아님)
- 프로젝트별, 브랜치별 자유롭게 조합 가능

### 5.6 스킬
- 특정 작업의 수행 방식 정의 (코드 리뷰 기준, 문서 포맷 등)
- 재사용 가능한 프리셋
- 프로젝트별 등록 및 적용
- 페르소나와 레이어 구분:
  - 페르소나: 에이전트의 성격/태도
  - 스킬: 작업 수행 방식

### 5.7 에이전트 관리
- 에이전트 실행 / 종료 / 재시작
- 프로젝트별 에이전트 할당
- 지원 엔진: claude / gemini / codex

### 5.8 입력창
- 마크다운 입력 + 프리뷰 (토글)
- 코드블록 (언어 지정, 하이라이팅)
- 파일 / 이미지 첨부
- 드래그앤드롭
- Shift+Enter 멀티라인

### 5.9 커맨드 체계

3개 레이어로 분리:

| | `/` 커맨드 | `!` 커맨드 | 스니펫 |
|---|---|---|---|
| **용도** | 앱 UI/기능 제어 | 에이전트/세션 제어 | 사용자 정의 프롬프트 단축키 |
| **예시** | `/new` `/branch` `/settings` | `!rt` `!persona` `!project` | `!정리` `!리뷰` `!의견` |
| **결과** | 앱 동작 | 에이전트/세션 동작 | 텍스트 → 에이전트 전송 |
| **관리** | 시스템 고정 | 시스템 고정 | 사용자 CRUD |
| **기존 tunapi** | 없음 | ✅ 이미 사용 중 | 없음 |

스니펫:
- 사용자가 직접 생성/편집/삭제
- 프로젝트별 또는 전역으로 관리
- 입력창에서 트리거 입력 시 자동완성

### 5.11 컨텍스트
- 프로젝트 내 대화 히스토리 유지
- 브랜치 merge 내용 main 컨텍스트에 축적
- 상세 구현은 tunapi 컨텍스트 고도화 참고

## 6. UI/UX

### 레이아웃 (데스크탑)
```
┌─────────┬──────────────────────┬─────────────┐
│ 사이드바  │ 채팅 메인             │ 컨텍스트 패널 │
│         │                      │             │
│ 프로젝트  │ [브랜치 탭]           │ 현재 프로젝트 │
│  └ main │                      │ 에이전트 상태 │
│  └ 브랜치│ 대화 영역             │ 페르소나     │
│  └ 브랜치│                      │ 스킬        │
│         │                      │             │
│ 설정     │ [입력창]              │             │
└─────────┴──────────────────────┴─────────────┘
```

### 레이아웃 (모바일)
- 사이드바: 스와이프 or 햄버거 메뉴
- 컨텍스트 패널: 하단 시트 (올려서 보기)
- 기본 상태: 채팅 영역만 표시

### 패널 토글
- 좌측 사이드바: 토글 가능
- 우측 컨텍스트 패널: 토글 가능
- 기본 상태(데스크탑): 둘 다 열림
- 좁은 창 or 모바일: 닫아서 채팅 집중

### 디자인 레퍼런스
- 구조: Slack / Mattermost (사이드바 + 채팅 패널)
- 대화 스타일: Claude.ai (문서형, 마크다운)
- 새로 필요한 것: 브랜치 탭, 토론 모드 진행 UI (턴 표시, 라운드 카운터), merge/abandon 액션
