# AGENTS.md

## Default Mode

- Treat this repository as read-only by default.
- Do not make code, config, service, or deployment changes unless the user explicitly asks for them.
- Prefer analysis, review, planning, and documentation-first work.

## Documentation Exception

- User-requested documentation work is allowed.
- You may create or update Markdown documents when the user explicitly asks for documentation, plans, notes, or reference material.
- Keep documentation changes scoped and avoid unrelated edits.

## Editing Guardrails

- If a task could affect runtime behavior, ask or wait for an explicit user request before changing code.
- When documentation refers to current implementation details, verify them from the repository first.
- Do not rewrite or delete existing user-authored plans unless explicitly requested.
